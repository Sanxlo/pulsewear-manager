Gestión del proyecto
Organización del trabajo

Para el desarrollo de PulseWear se utilizó un enfoque basado en metodologías ágiles. La planificación y seguimiento de tareas se realizó mediante un tablero Kanban en Trello.

El objetivo principal fue dividir el proyecto en funcionalidades pequeñas y manejables para facilitar el desarrollo progresivo y la detección temprana de errores.

Herramienta utilizada

Se utilizó Trello como herramienta de gestión visual del proyecto.

El tablero se organizó en las siguientes columnas:

Backlog

Contiene todas las funcionalidades, mejoras y tareas identificadas para el proyecto.

Ejemplos:

Crear catálogo de productos.
Implementar carrito.
Crear checkout.
Desarrollar API REST.
Diseñar panel de administración.
Todo

Tareas listas para comenzar.

Ejemplos:

Crear ProductCard.
Implementar filtros del catálogo.
Crear rutas principales.
In Progress

Tareas actualmente en desarrollo.

Ejemplos:

Integración del carrito.
Desarrollo del comparador.
Gestión de pedidos.
Review

Tareas completadas pendientes de revisión y pruebas.

Ejemplos:

Verificación de formularios.
Revisión responsive.
Validación de la API.
Done

Tareas finalizadas y verificadas correctamente.

Ejemplos:

Navbar.
Hero Section.
Catálogo.
Checkout.
Administración de pedidos.
División de funcionalidades

Cada funcionalidad principal se dividió en subtareas técnicas.

Catálogo
Crear página CatalogPage.
Obtener productos desde la API.
Implementar búsqueda.
Implementar filtros por categoría.
Crear ProductCard reutilizable.
Producto
Crear ProductDetailPage.
Mostrar imagen y detalles.
Añadir selección de talla.
Implementar productos relacionados.
Carrito
Crear CartContext.
Añadir productos.
Eliminar productos.
Calcular total.
Checkout
Crear formulario de envío.
Crear formulario de pago.
Validar datos.
Crear pedido mediante API.
Administración
Mostrar pedidos.
Filtrar pedidos.
Buscar clientes.
Cambiar estados.
Eliminar pedidos recibidos.
Backend
Crear arquitectura por capas.
Implementar rutas.
Implementar controladores.
Implementar servicios.
Crear almacenamiento de pedidos.
Flujo de trabajo

Durante el desarrollo se siguió el siguiente proceso:

Crear una tarea en Backlog.
Mover la tarea a Todo.
Desarrollar la funcionalidad.
Mover la tarea a In Progress.
Revisar funcionamiento y corregir errores.
Mover la tarea a Review.
Una vez validada, moverla a Done.
Beneficios obtenidos

La utilización de Trello y Kanban permitió:

Mantener una visión clara del estado del proyecto.
Dividir tareas complejas en partes más pequeñas.
Detectar bloqueos rápidamente.
Priorizar funcionalidades importantes.
Controlar el progreso general del desarrollo.
Conclusión

La organización mediante Trello y Kanban facilitó el desarrollo de PulseWear, permitiendo gestionar de forma visual todas las funcionalidades del frontend, backend y documentación. Gracias a esta planificación fue posible desarrollar el proyecto de forma ordenada y mantener un seguimiento constante del progreso.
